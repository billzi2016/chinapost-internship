## 第九部分：EMS 在线客服知识库（imcloud）核心业务流与拦截/改单动态政策数据

> 说明：本文件中涉及在线客服、门户系统、后台工单、接口返回等内容，多属于经验性整理，并不等同于公开发布的统一制度。涉及具体拦截条件、费用、错误码和风控规则时，应以实时系统提示和客服正式回复为准。

基于分布式客服智能云系统（imcloud）的底层逻辑，机器人需集成以下关于邮件在途状态变更、拦截及地址修改的强时效性业务规则：

### 9.1 邮件在途拦截（Interception）刚性判定政策

- **可拦截状态区间：** 一般只有在邮件尚未进入末端派送、且仍处于可调度链路中时，系统才可能受理拦截申请。
- **不可拦截阶段：** 一旦邮件已经进入投递、妥投或接近签收阶段，拦截成功率通常明显下降，很多情况下会转为拒收或退回处理。
- **拦截计费与退回路径：**
- **同城拦截：** 部分场景下处理成本较低，但是否收费仍取决于产品和网点执行口径。
- **跨省/跨干线拦截：** 若邮件已进入跨区域运输链路，通常更容易产生额外处理成本，原寄邮资是否退还、是否补收费用，以实际受理口径为准。

### 9.2 在途地址修改（Address Modification）与转寄政策

- **修改权限：** 邮件在未妥投前，原始寄件人通常更容易发起修改地址或联系方式的申请；收件人自行改单时，往往需要额外身份核验或寄件方确认。
- **费用浮动红线数据：**
- **同区域改投（同网点/同区县）：** 同区域内调整通常更容易处理，是否收费以及顺延时效，以具体网点和系统判断为准。
- **跨区县/跨地市改投：** 跨区域改投往往需要重新调度，可能产生补费、时效延长或退回重寄等结果。

## 第十部分：EMS 协议客户门户（PCP）结算、账期、API 错码与风控红线

针对大客户、电商卖家在 `my.ems.com.cn` 门户内高频触发的账户、API 接口及账期争议，机器人知识库须具备以下底层后台政策：

### 10.1 协议客户账期与熔断（Freeze）风控政策

- **预存透支额度控制：** 预付费协议客户余额过低时，系统通常会触发预警；余额不足或欠费严重时，可能暂停面单下发或批量出单能力。
- **后付费账期红线：** 后付费客户一般需要在账单周期内完成核对和付款。逾期后，常见后果包括催缴、权限暂停、附加费用或人工审核加强，但具体规则以合同和门户口径为准。

### 10.2 PCP 门户面单打单（API）异常状态码政策数据

当客户的 ERP/WMS 系统对接中国邮政 PCP 开放平台返回错误时，客服机器人必须能直接回传底层政策判定：

| API 错误代码 | 系统底层触发状态原因 | 对应风控/前台处理政策 |
| --- | --- | --- |
| **认证类错误** | 企业密钥失效、权限不足、协议异常或账户状态异常 | 通常需要检查账号、合同状态或欠费情况 |
| **地址类错误** | 地址不规范、受限区域、风控拦截或禁寄属性触发 | 需要校验地址、品类或人工复核 |
| **流量类错误** | 并发过高、频率过快或接口限流 | 通常需要稍后重试或联系技术支持 |

## 第十一部分：11185 综合邮政平台跨板块联动与特种寄递政策

`11185.cn` 承载了除了快递（EMS）之外的普通邮政、集邮、报刊征订及邮政金融跨板块联动业务，知识库必须包含以下交叉服务政策：

### 11.1 集邮商品（Philatelic Products）特种配送与保价限制

- **损毁界定特异性：** 邮票、首日封、珍藏册等集邮商品若发生水湿、折角、污损、齿孔断裂，即视为**全损（失去全部收藏价值）**。
- **保价与赔付边界：** 对于珍稀集邮商品，保价和赔付通常更强调票面价值、发售凭证或正式购买凭证，不会简单按二级市场炒作价格计算。

### 11.2 报刊订阅（Periodicals Subscription）退订与变更政策

- **退订时效截止线：** 报刊订阅的中途退订通常存在申请截止时间，且往往需要在后续发运或结算节点前提出。
- **退款计算公式：**

$$退款金额 = 原始总订阅价 - 已发生的投递成本 - 相关渠道或服务费用$$

已起运、已投递到户的报刊概不接受退还和退款。

## 第十二部分：国家邮政局（SPB）申诉时效与“有效申诉”认定判定矩阵

当用户在 `zwfw.spb.gov.cn`（国家邮政局政务服务门户）发起最终行政申诉时，中国邮政内部处理机器人及前台客服必须执行以下应对与定责政策：

### 12.1 消费者发起行政申诉的刚性前置条件

国家邮政局申诉网站不予受理未经过企业内部投诉的直诉。用户必须同时满足以下两项政策红线：

1. **企业处理前置：** 用户通常需要先向中国邮政/EMS 官方客服渠道发起正式投诉；只有在企业处理期内未妥善答复，或者用户对处理结果明显不认可时，才更容易进入行政申诉阶段。
2. **事件时效跨度：** 发起申诉的时间距离邮件交寄之日，**国内邮件不得超过 1 年，国际邮件不得超过 180 天**。

### 12.2 中国邮政对国家邮政局转办件的“限时办结”政策

- **限时办结要求：** 国家邮政局转办件通常要求企业在较短周期内完成调查、回复和系统回填。
- **逾期升级处罚：** 若长期未办结或处理明显不当，企业可能面临监管通报、处罚或服务质量考核压力，但具体处罚口径取决于适用法规和案件情形。

### 12.3 行业核心指标：“有效申诉率（Valid Complaint Rate）”内部定责红线

并非所有消费者的申诉都会被国家邮政局判定为邮政企业的责任。海关查验、不可抗力等引发的投诉，邮政企业需在后台上传证明文件申请冲销。

## 第十三部分：客服机器人高频极端客诉意图（Intent）及刚性话术判定树数据

为满足客服机器人底层逻辑配置，针对极度敏感的客诉场景，制定以下无缝衔接系统判定树与刚性法律/政策回复规范：

### 13.1 意图：投诉“邮政包裹放驿站/快递柜未送货上门”

- **触发条件：** 物流显示“已签收（智能快件箱/代收点）”，用户输入包含“未允许”、“投诉”、“没送上门”。
- **底层政策判定：** 违反《快递市场管理办法》第二十八条（经营快递业务的企业未经用户同意，不得代为确认收到快件，不得擅自将快件投递到智能快件箱、快递服务站等快递末端服务设施）。
- **系统动作流：**
1. 立即停止自动机器人话术。
2. 后台通常会优先转人工介入，并向末端投递环节发起加急核查或补投指令。
3. 若涉及快件箱、驿站或代收点，系统可能进一步核对授权状态，并协调末端服务主体配合处理。

### 13.2 意图：国际进境邮件显示“送交海关（Held by Customs）”超 72 小时无更新

- **触发条件：** 跟踪轨迹在“国内互换局”或“海关”状态停滞，用户情绪激化，输入“扣留”、“为什么不放行”、“缴税”。
- **底层政策判定：** 海关法属于国家司法管辖。邮政企业仅扮演“物理运输与代递单证”角色，无权干涉海关查验流转速度。
- **机器人拦截应答的核心政策数据话术：**

> “尊贵的客户，根据《中华人民共和国海关法》及邮政普遍服务协议，您的国际邮件（单号：[面单号变量]）目前正处于海关法定查验段。此阶段属于海关行政执法流程，非邮政物流分拨网络。请您密切关注由【中国邮政】发送的关税申报短信，或登录‘掌上海关’App 凭邮件号据实进行补充申报或税款缴纳。邮政公司在海关放行前，依法不具备单方面提取或加速配送的权限。”
