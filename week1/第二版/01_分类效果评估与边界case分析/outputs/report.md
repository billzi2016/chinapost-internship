# 分类效果评估与边界 case 分析报告

## 1. 分析目的

第一版中使用 `gpt-oss:20b` 对 CSDS 对话做“快递 / 邮政相关”二分类。反馈要求补充说明：已有二分类结果是否做过探查、整体情况如何、是否存在边界 case、是否有量化指标评估分类效果，并建议用 Regex 分析总结字段中的关键词来打弱标签。

本报告采用同一宽口径任务定义进行分析：

> 判断给定对话是否和快递、物流、配送、邮政、EMS、运费、签收、揽收、退费、地址修改等相关。

分析链路为：

1. 保留已有 `gpt-oss:20b` 二分类结果，不重新改动。
2. 使用 Regex 在总结字段中构造可解释弱标签。
3. 使用 `gpt-oss:120b` 按同一二分类定义独立复核，并额外输出业务精细类别。

## 2. 当前已完成结果

当前已完成的是 20B 与 Regex 的对比。新的 120B 宽口径复核正在按同一任务定义重新生成，旧的窄口径“严格邮政 / EMS”结果不再作为本报告依据。

## 4. 20B 与 Regex 对比

| 标签组合 | 样本数 | 占比 |
| --- | ---: | ---: |
| 20b=True regex=True | 5102 | 47.68% |
| 20b=True regex=False | 1219 | 11.39% |
| 20b=False regex=True | 789 | 7.37% |
| 20b=False regex=False | 3591 | 33.56% |

![20B 与 Regex 对比](figures/20b_vs_regex_matrix.png)

20B 与 Regex 同时判为相关的样本有 5102 条，占全量 47.68%。这部分通常包含快递、物流、配送、取件、签收、站点、运费等显式业务词，是当前二分类结果中最容易解释的一类样本。

边界 case 主要来自两类：

1. `20b=True regex=False`：20B 判为相关，但总结字段没有命中 Regex，可能是模型根据完整对话上下文识别出的隐性物流场景，也可能是误判。
2. `20b=False regex=True`：Regex 命中，但 20B 判为不相关，可能是关键词出现在发票、售后、安装、会员等非寄递主题中，也可能是 20B 漏判。
