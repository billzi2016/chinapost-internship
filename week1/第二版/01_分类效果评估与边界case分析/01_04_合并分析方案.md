# 01/04 合并分析方案：20B 结果评估 + Regex 弱标签 + 120B 精细分类

## 当前有效口径

本分析只使用一个二分类定义：

> 判断给定对话是否和快递、物流、配送、邮政、EMS、运费、签收、揽收、退费、地址修改等相关。

该定义来自第一版 `gpt-oss:20b` 过滤任务。后续 Regex 和 120B 都围绕这个宽口径定义服务。

## 分析目标

1. 保留已有 20B 二分类结果，不重新改动。
2. 用 Regex 在总结字段中构造可解释弱标签，观察其与 20B 结果是否接近。
3. 用 120B 按同一二分类定义独立复核，并额外输出业务精细类别。
4. 将业务精细类别用于可视化标签和聚类解释。

## 120B 输出字段

```json
{
  "broad_related": true,
  "category": "时限时效类",
  "reason": "用户询问快递预计送达时间。",
  "confidence": 0.95
}
```

字段说明：

- `broad_related`：是否符合统一宽口径二分类定义。
- `category`：业务精细类别。
- `reason`：一句话说明依据。
- `confidence`：置信度，用于人工复查排序。

## 精细类别

120B 从以下类别中选择一个：

| 类别 | 说明 |
|---|---|
| 费率价格类 | 运费、邮费、收费、报价、保价等 |
| 时限时效类 | 多久到、什么时候送达、延误、派送进度等 |
| 流程规则类 | 地址修改、签收、拒收、退回、取件、站点等流程问题 |
| 禁限寄类 | 禁寄、限寄、违禁、液体、电池、药品等 |
| 国际业务类 | 国际、海外、清关、海关、跨境等 |
| 泛物流配送类 | 宽口径物流配送相关，但不适合归入以上细类 |
| 非邮政相关 | 不符合宽口径相关定义 |

## 当前状态

- Regex 结果已完成，规模与 20B 接近。
- 旧 120B 窄口径结果已废弃，不再作为报告依据。
- 新 120B 宽口径复核需重新跑完后再生成最终报告。
