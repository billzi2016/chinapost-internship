# 03 代码仓库管理说明

## 对应反馈

处理代码建议建立 GitHub 仓库，统一管理。

## 计划解决内容

- 梳理当前代码、报告、数据、输出文件的目录职责。
- 给出适合提交到 GitHub 的仓库结构建议。
- 明确哪些文件应纳入版本管理，哪些数据和中间产物应通过 `.gitignore` 排除。
- 整理后续建仓、提交、README、requirements 的最小方案。

## 当前注意事项

- 未经确认不执行任何 git 操作。
- 不主动 `git init`、`git add`、`git commit`、`git push`。
- 数据文件较大或包含原始语料时，应优先排除出公开仓库。

## 预期输出

- 仓库结构建议
- `.gitignore` 建议
- GitHub 仓库 README 大纲
- 代码统一管理说明
