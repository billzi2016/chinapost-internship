# 02 模型选型补充

## 对应反馈

大模型选型需要进一步完善，从训练效率、量化后内存占用、CPU 推理速度、中文语义理解能力、长对话上下文建模、JSON 结构化输出准确率、商用合规性等角度补充，最好给出类似“模型文件大小 3.2GB，运行时内存占用 4.5GB，单条推理 xx 秒”的信息。

## 计划解决内容

- 在原模型选型报告基础上补充工程落地维度。
- 建立候选模型对比表，覆盖 Qwen2.5、gpt-oss:20b 及其他候选路线。
- 将模型选择理由从“能力描述”扩展为“训练、推理、部署、合规、结构化输出”的综合判断。
- 如本机已具备模型运行环境，再补本机实测文件大小、内存占用、推理耗时。
