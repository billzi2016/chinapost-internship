## 当前结论摘要

当前阶段的选型结论可以概括为三句话：

1. 主力 SFT 候选优先放在 `Qwen2.5-3B` 和 `Qwen2.5-7B`
2. `gpt-oss:20b` 可以作为更大模型上限和本地强基线，但不建议作为第一优先 SFT 主力
3. `70B` 级别模型、`Llama 4` 多模态 MoE、以及明显偏 `thinking / reasoning` 的路线，都不适合本项目当前的数据规模、中文重点和实习阶段资源约束

## 为什么结论会偏向 Qwen2.5

对这个项目来说，模型最重要的不是“理论上最强”，而是“在当前数据量、中文场景、SFT 难度和本地资源下最稳妥地做出结果”。  
因此，模型选型优先级是：

- 中文能力
- 指令跟随能力
- 结构化输出能力，尤其是 `JSON`
- 小数据下做领域适配的稳定性
- 推理速度和部署可行性
- 是否容易完成从训练到验证到展示的闭环

从这几个维度看，`Qwen2.5` 比很多最新但更重、更复杂、或更偏 reasoning 的模型更符合当前阶段目标。
