# 词云与可视化

## 1. 这个模块在项目里做什么

`stats/vis.py` 负责把前面得到的统计结果和关键词结果转成图片，方便观察和汇报。  
它做的不是新的文本算法，而是**把已有结果可视化**。

对应代码位置：

- `/Users/bizi/Desktop/邮政实习/week1/stats/vis.py`

## 2. 本项目里有哪些图

当前主要生成两大类图：

### 2.1 基础统计图

包括：

- 对话轮数直方图
- 对话总 `token` 数直方图
- 对话总词数直方图
- 高频词 `Top20` 柱状图
- 普通词云

这些图的作用是：

- 看长度分布
- 看高频词
- 快速形成对语料风格的感性认识

### 2.2 关键词方法图

针对三种关键词方法：

- `TF-IDF`
- `TextRank`
- `KeyBERT`

分别生成：

- `Top20` 柱状图
- 方法词云

这些图的作用是：

- 直观看每种方法选出来的关键词差异
- 对比不同方法更偏重哪些词

## 3. 直方图的原理和做法

### 3.1 原理

直方图用来展示一个数值型变量的分布。  
比如：

- 一条对话有多少轮
- 一条对话有多少 token

它的思路是：

- 先把数值划分到不同区间
- 再统计每个区间里有多少样本

这样就能看出：

- 数据是集中在短对话，还是长对话
- 有没有特别长的尾部

### 3.2 本项目里怎么做

项目里统一使用 `matplotlib.pyplot.hist` 画图，  
每张图固定：

- `figsize=(10, 6)`
- `bins=30`

并统一设置：

- 标题字号更大
- 横纵轴字号更大
- 刻度字号更大

这是为了满足汇报图可读性要求。

## 4. 高频词柱状图的原理和做法

### 4.1 原理

柱状图适合展示离散对象的数量比较。  
在这里，对象就是“词”，数值就是“出现次数”或“关键词得分”。

### 4.2 本项目里怎么做

对于普通高频词图，代码会：

1. 对 `words` 做 `Counter`
2. 取前 `20` 个最高频词
3. 用 `plt.bar` 画柱状图

对于方法关键词图，代码会：

1. 取该方法前 `20` 个关键词
2. 横轴放词
3. 纵轴放分数或出现次数

这样一来，不同方法都能用统一的图形模板展示。

## 5. 普通词云的原理和做法

### 5.1 原理

词云本质上是把词的权重映射到：

- 字体大小
- 空间布局

词越重要或越高频，通常显示得越大。  
它不是精确统计图，而是更偏直观展示。

### 5.2 本项目里怎么做

普通词云使用的是：

- 全部原始 `words`

也就是说，权重本质上来自词频。  
代码里通过：

- `WordCloud.generate(" ".join(words))`

直接从文本生成词云。

为了避免中文乱码，项目会先自动寻找系统里的中文字体。  
如果没有找到合适字体，就跳过词云生成。

## 6. 方法词云的原理和做法

### 6.1 和普通词云有什么区别

普通词云看的是：

- 词出现多少次

方法词云看的是：

- 某种关键词方法给这个词打了多少分

因此，方法词云不是频率视角，而是“算法认为它有多重要”的视角。

### 6.2 本项目里怎么做

项目会先把关键词列表变成一个字典：

- `token -> score`

然后调用：

- `WordCloud.generate_from_frequencies(frequencies)`

这样画出来的词云中：

- 分数越高的词越大

这套逻辑统一用于：

- `TF-IDF`
- `TextRank`
- `KeyBERT`

## 7. 字体和显示处理

中文可视化里一个现实问题是乱码。  
因此项目里专门做了字体自动查找，候选字体包括：

- `STHeiti`
- `PingFang`
- `Hiragino Sans GB`
- `Songti`
- `Arial Unicode`

只要找到本机存在的字体，就用于：

- 词云
- `matplotlib` 中文显示

同时还设置了：

- `axes.unicode_minus = False`

避免坐标轴负号显示异常。

## 8. 为什么这些可视化适合本项目

### 8.1 适合做汇报

纯表格很难快速说明数据特征。  
这些图可以直接回答：

- 对话普遍长不长
- 高频主题词有哪些
- 不同关键词算法怎么看这个语料

### 8.2 能把方法差异展示出来

如果只输出文本列表，很难直观看出：

- `TF-IDF` 偏什么
- `TextRank` 偏什么
- `KeyBERT` 又偏什么

统一做成 `Top20` 图和词云后，差异会非常直观。

### 8.3 有利于发现异常

例如：

- 词云里是否仍然残留很多虚词
- 高频词图里是否被某些模板化表达占满
- 长度分布里是否存在异常长对话

这些都能反过来帮助检查数据和方法。

## 9. 局限

### 9.1 词云偏展示，不够精确

词云适合汇报和感性观察，但不适合做精细定量分析。  
真正的比较还是应该看表格或具体分数。

### 9.2 可视化质量依赖前面的数据质量

如果前面关键词提取得不好，  
图再漂亮也只是把不好的结果可视化出来。

### 9.3 图只能展示结果，不能替代解释

所以在正式汇报里，图和方法说明必须配套出现。  
这也是为什么需要单独写这套知识库文档。
