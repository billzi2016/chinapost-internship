# Rebuild Notes

The current repository is not a direct copy of the original company-machine project. It is a rebuilt and reorganized version.

The reason is straightforward: the original development environment, training artifacts, and part of the working data could not be exported in full. What remains here is a reconstruction based on the already completed project structure.

This rebuild mainly means:

1. Code and documents were reorganized into a readable structure.
2. Non-portable data and environment-specific artifacts were removed.
3. The `week1`, `week2`, and `week3` labels reflect project phases only and do not correspond to real calendar weeks.

Time spent on manual tuning, sample filtering, validation-dataset preparation, onboarding training, and internal meetings has already been removed from this staged naming, so these `week` labels should be read as task phases rather than any real-world timeline.
