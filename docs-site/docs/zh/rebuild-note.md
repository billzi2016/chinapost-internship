# 重建说明

当前仓库不是把公司电脑上的原始项目完整拷贝出来的版本，而是一次重建整理。

原因很直接：原始开发环境、训练产物和部分业务数据没有办法完整迁出，所以现在这套仓库更接近“按已完成结果重建”，而不是“把原始工程原封不动搬过来”。

这里的重建主要体现在三个方面：

## 1. 代码和文档重新归档

原始项目推进时，很多内容分散在不同目录、不同阶段和不同机器环境里。当前仓库做的事情，是把这些材料重新整理成一个可读、可解释、可继续维护的结构。

## 2. 数据与环境做了收敛

有些原始数据、训练中间产物和部署上下文无法直接带出，所以当前版本保留的是：

1. 结构化后的代码与文档。
2. 可以公开保留的实验结论。
3. 可以继续复现和说明的工程入口。

## 3. 时间线按任务阶段重排

仓库中的 `week1`、`week2`、`week3` 以及后续阶段名称，主要对应任务推进顺序，不与真实自然周严格一一对应。

这里没有把人工调参、样本筛选、验证数据集整理、入职培训、会议沟通等时间单独计入阶段命名，因此这些 `week` 更适合理解为任务阶段。
